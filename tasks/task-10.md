угадывать:
```
worker=<id> ok=true workspace=<abs path>
files:
- <relpath> -> <abs path>
message:
...
```

## 2. DRIVER_SYSTEM — дополни тремя правилами (кратко, в стиле имеющихся)
- Директории воркеров в runs/ никогда не удаляются; пути из workspace= и
  files: результата spawn_worker используй буквально.
- Не нашёл файл — ищи (find/ls), не предполагай.
- Если проверить результат не удалось — напиши об этом в finish(result)
  прямо («проверка не удалась, потому что...»). Выдумывать причины
  недоступности запрещено.

## 3. Тесты
