# T5: Приёмочные сценарии A1-A3 (чекеры)

Контекст: T1-T4 приняты. Это кодификация приёмки из SPEC.md. Сценарии —
обычные python-скрипты, НЕ pytest. Они гоняют живую систему и проверяют
реальные артефакты. Запрещено что-либо мокать; чекер смотрит только на
файлы рана, stdout и exit code.

## Что сделать

### 1. `scenarios/common.py`
Хелперы (минимум): запуск `uv run swarm run <task>` сабпроцессом с
env-переопределениями, парс `run_dir=` из stdout, чтение transcript.jsonl,
подсчёт tool-вызовов по именам. Без классов-фреймворков — функции.

### 2. `scenarios/a1_code_selfcheck.py`
- Задача (дословно): «Создай python-пакет strcalc: модуль с функцией
  evaluate(expr: str) -> float, которая безопасно вычисляет арифметические
  выражения вроде '2+3*4' (должно дать 14). Напиши pytest-тесты минимум на
  4 случая включая приоритет операций и деление. Прогони тесты и убедись,
  что они зелёные, прежде чем завершать.»
- Чекер: exit code 0, status=completed в stdout; в run_dir СВОИМ сабпроцессом
  `python3 -m pytest -q` из корня рана → exit 0; `evaluate("2+3*4") == 14.0`
  отдельным `python3 -c` из корня рана.

### 3. `scenarios/a2_decomposition.py`
- Задача: «Делегируй воркеру создание файла data.json — JSON-массив из 5
  целых чисел на его выбор. После его завершения делегируй ВТОРОМУ воркеру:
  передай ему числа и попроси записать их сумму в файл sum.txt в его
  директории. Затем сам проверь, что сумма верна, и заверши.»
- Чекер: status=completed; в run_dir/workers/ ≥2 непустых workspace;
  data.json существует и парсится в массив из 5 int; sum.txt существует и
  int(содержимое) == сумме массива (чекер пересчитывает сам); в transcript
  суммарно ≥2 вызовов spawn_worker/spawn_workers.

### 4. `scenarios/a3_budget_stop.py`
- env: `SWARM_MAX_ITERATIONS=3`.
- Задача заведомо больше бюджета: «Напиши полноценный статический генератор
  сайтов: парсер markdown, шаблонизатор, инкрементальная сборка, CLI,
  тесты на всё.»
- Чекер: exit code 3; stdout содержит `status=budget_exceeded`; в
  transcript.jsonl ≤ 25 строк (бюджет реально остановил); процесс уложился
  в 10 минут (timeout на сабпроцесс, по истечении — FAIL «завис»).

### 5. `scenarios/run_all.py`
Запускает a1, a2, a3 по очереди, печатает `<имя>: PASS|FAIL <причина>`,
exit 1 если хоть один FAIL. Никакой магии, никакого parallel.

## Критерии готовности
1. `uv run ruff check src tests scenarios` чисто; `uv run pytest -q`
   по-прежнему зелёные (scenarios не подхватываются pytest'ом).
2. Живой прогон `uv run python scenarios/run_all.py` — все три PASS.
   Вставь полный фактический вывод в финальное сообщение. Если какой-то
   сценарий FAIL — НЕ подгоняй чекер под фактическое поведение и не
   ослабляй ассерты: доложи как есть, с выводом чекера и хвостом
   transcript.jsonl проблемного рана.
3. Один коммит `feat: acceptance scenarios a1-a3`.

## Запрещено
Изменения в src/ (если сценарий вскрыл баг ядра — доложи, не чини);
pytest-обёртки вокруг сценариев; моки; снижение строгости чекеров ради PASS.
