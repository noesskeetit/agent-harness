## Фаза 2: реализация (если разведка ок)

### 1. `worker.py`
```python
def spawn_manus_worker(
    task: str,
    worker_id: str,
    run_dir: Path,
    config: Config,
) -> WorkerResult: ...
```
- Тот же контракт, что у codex-бэкенда: workspace
  `run_dir/workers/<worker_id>/`, `task.md`, служебное в `.worker/`
  (лог процесса), snapshot-дифф файлов, таймаут+killpg, честные фейлы.
- env: `LLM_API_KEY` = config.api_key, base-переменная по образцу адаптера
  (для FM API), thinking off, плюс прокси-блок НЕ нужен (FM API ходит
  напрямую, см. NO_PROXY-практику в старом харнессе).
- Запуск upstream-рантайма из `~/Code/manus-agent` тем способом, который
  выяснишь в разведке (его venv/uv — не тащи зависимости в наш pyproject).
- Финальное сообщение воркера: возьми из того файла, который Manus реально
  пишет (summary.md или эквивалент); нет файла → ok=False с пояснением.

### 2. Выбор бэкенда в тулах
- `spawn_worker` и элементы `spawn_workers` получают опциональный параметр
  `backend: "codex" | "manus"` (дефолт `"codex"`). Неизвестный backend →
  ошибка тула текстом. Описание в схеме: «manus — автономный
  исследовательский агент; codex — кодинг-агент».

### 3. Тесты
Стаб-исполняемый как в T3 (для manus-пути — стаб, имитирующий выходной
файл результата). Поведение: успех / нет файла результата / таймаут /
неизвестный backend через тул. Реальный Manus в юнитах не зовём.

### 4. Живой прогон
`uv run swarm run "делегируй воркеру с backend=manus задачу: создай в своей
директории файл note.md с тремя фактами о Kaggle. Проверь файл и заверши"`.
Если живой Manus не заведётся с разумных попыток — доложи лог, не хакай.
