# Инструкции для исполнителя (Codex)

Ты — исполнитель в паре «оркестратор (Claude) — исполнитель (ты)». Тебе дают
ТЗ из `tasks/task-NN.md`; оркестратор ревьюит твой дифф после каждой задачи.

## Жёсткие правила

1. Прочитай `SPEC.md` перед первой задачей. Анти-bloat правила из него —
   закон: никаких новых абстракций, реестров, валидаторов, gates, audit-файлов
   сверх явно запрошенного в ТЗ.
2. Делай РОВНО то, что в ТЗ. Заметил проблему вне скоупа — напиши в финальном
   сообщении, не чини сам.
3. Не создавай документацию, отчёты, README, примеры, если ТЗ не просит.
4. Не добавляй зависимости сверх перечисленных в SPEC.md.
5. Тесты пиши на поведение. Запрещены: ассерты строк в промптах, точных
   последовательностей шагов, наличия ключей в dict ради наличия.
6. Заверши задачу коммитом: `feat|fix|chore: <суть>` (одна задача = один
   коммит). В коммит не включай `.env`, `runs/`, `.agent/`.

## Команды

```bash
uv sync                 # окружение
uv run pytest -q        # тесты — должны быть зелёные перед коммитом
uv run ruff check src tests   # линт — чисто перед коммитом
uv run swarm <cmd>      # CLI
```

## Финальное сообщение (его читает оркестратор)

Структура: что сделано (1-3 строки) → как проверял (команды + фактический
вывод) → что заметил вне скоупа (если есть). Без воды и самопохвалы.
